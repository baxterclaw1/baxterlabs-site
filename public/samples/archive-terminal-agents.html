Under the Hood

Anatomy of an Agent Brain

Every agent is built from the same modular blueprint. Markdown files define who it is. Skills define what it can do.

01 IDENTITY.md

Who the agent is — its name, role, personality traits, and how it introduces itself to the world.

02 CONSTITUTION.md

Hard rules and ethical boundaries. What the agent must never do, regardless of what it thinks is optimal.

03 SOUL.md

Voice, tone, and values. How the agent communicates — its writing style, humor level, and personality.

04 MEMORY.md

Persistent knowledge that grows over time. Daily logs, observations, ideas, and learned patterns.

05 skills/

Modular capabilities. Each skill is a self-contained directory with its own instructions, scripts, and tools.

06 HEARTBEAT.md

Self-check routine. The agent reports its own health, active tasks, and blockers every cycle.

07 cron-jobs.json

Scheduled tasks. Research cycles, data collection, nightly retrospectives — the agent's daily rhythm.

Principles

How We Build

"Reliable first, clever second."

We don't ship impressive demos. We ship agents that run for weeks without intervention. Uptime beats elegance.

"Minimum viable architecture."

Every component earns its place. If a skill goes unused for 90 days, it gets removed. The right complexity is the minimum needed now.

"Evidence over speculation."

Changes come from runtime data and agent retros, not guesses about what might work. Every decision has a receipt.
